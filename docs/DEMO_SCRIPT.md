# Demo Script (Interviews)

This is a 5–8 minute walkthrough designed for calm narration.

## 0) Setup (optional)

- Start from a fresh install (or clear app data) so onboarding appears.
- If onboarding does not show (existing install), open it from Settings → Help & About → Onboarding walkthrough.

## 1) First launch → onboarding (30–60s)

- Show the onboarding screens.
- Emphasize the scope: track transactions, plan ahead with scheduling, budgets as soft guidance, analytics for visibility, and data ownership/export.
- Tap **Skip** once to demonstrate it’s always optional.

## 2) Add a transaction (60–90s)

- Go to **Transactions**.
- Tap **Add**.
- Create an **Expense**:
  - Pick an account
  - Choose a category
  - Enter amount and date
  - Save
- Call out that expenses/income/transfers are first-class and stay local.

## 3) Budget → soft warning (60–90s)

- Go to **Budgets**.
- Create a budget for a category with a modest amount.
- Add another expense in that category to approach or exceed the budget.
- Show the budget state and explain: it’s a soft warning (no blocking).

## 4) Analytics → charts (60–90s)

- Open **Analytics**.
- Point out the trend view / category breakdown.
- Mention: analytics is computed locally from stored transactions.

## 5) Scheduled / recurring execution example (60–90s)

- Add a **scheduled** transaction dated in the near future.
- Add a **recurring** transaction (e.g., monthly).
- Explain the execution pipeline:
  - Runs on app launch and on resume
  - Executes due scheduled items, then recurring
  - No background jobs/notifications

## 6) Export data (45–60s)

- Go to **Settings** → **Export data**.
- Tap **Generate & share**.
- Show the system share sheet.
- Explain what’s inside:
  - UTF‑8 CSVs with headers
  - ISO date format
  - Amounts in major units
  - Bundled as a ZIP

## Close (10–20s)

- Summarize: local-first finance app, calm UI, core flows covered, portable export, and lightweight scheduling execution.
